Find the K-Sum of an Array

Medium BFSArrayStringGreedyHeap

Find the K-Sum of an Array Example 1:

Input: nums = [2,4,-2], k = 5 Output: 2 Explanation: All the possible subsequence sums that we can obtain are the following sorted in decreasing order: 6, 4, 4, 2, 2, 0, 0, -2. The 5-Sum of the array is 2. Example 2:

Input: nums = [1,-2,3,4,-10,12], k = 16 Output: 10 Explanation: The 16-Sum of the array is 10.

Constraints:

n == nums.length 1 <= n <= 10^5 -10^9 <= nums[i] <= 10^9 1 <= k <= min(2000, 2^n)

Short script to say to the interviewer

‚ÄúFirst compute the maximum subsequence sum base (sum of all positive elements). Every other subsequence sum equals base - s where s is the sum of some subset of the absolute values of all elements (those we ‚Äòdrop‚Äô from base). So we need the k smallest subset sums s (starting from 0) of the array abs(nums). Since k ‚â§ 2000 we can enumerate the k smallest subset sums using a min-heap / BFS-on-combinations technique and return base - s_k.‚Äù

---

Intuition / transformation

  • Let base = sum(max(0, nums[i])). That's the largest possible subsequence sum (take every positive number, ignore negatives).
  • For each original element, define a[i] = |nums[i]|. Any subsequence sum equals base - t where t is the sum of absolute-values of elements we flipped/removed (i.e., choosing not to take a positive element or taking a negative element).
  • We need the k largest subsequence sums ‚Üí equivalently the k smallest values of t. The smallest t is 0 (taking the base set). So enumerate smallest t in increasing order and output base - t_k.
  • To generate the k smallest subset sums of a non-negative array a[], sort a ascending and use a min-heap that explores next sums by adding next elements ‚Äî classic technique similar to merging k sorted lists / BFS on combination space. Because k is small, this is efficient.

---

Algorithm (sketch)

  1. Compute base = sum of positive nums.
  2. Build array a of absolute values |nums[i]|, sort a in ascending order.
  3. Use a min-heap of nodes (sum, index) where sum is a subset-sum formed using elements among a[0..index] with the _last included element_ being a[index].
  • Initialize: push (a[0], 0). The subset-sum 0 (empty subset) is considered first (that gives base).
  1. Repeatedly pop the smallest sum from heap. For a popped node (s,i):
  • This s is the next smallest non-zero subset-sum candidate.
  • To generate further states:
  • If i + 1 < m, push (s + a[i+1], i+1) ‚Äî extend current subset by next element.
  • Also push (s - a[i] + a[i+1], i+1) ‚Äî replace last chosen a[i] by a[i+1] (this enumerates different combinations uniquely).
  1. Repeat until you have enumerated k subset sums (including the 0 sum at start). The answer is base - t where t is the k-th smallest subset-sum enumerated (first is 0 ‚Üí answer base).
  2. Use long for sums.

---

Complexity

  • Sorting a: O(n log n).
  • Heap operations: we perform at most O(k) pops and O(k) pushes. Each heap op is O(log k). So O(k log k).
  • Overall: O(n log n + k log k). For constraints given (k ‚â§ 2000), this is efficient.
  • Space: O(n) for a plus O(k) heap.

---

Correctness notes

This heap-generation technique is a standard way to enumerate the smallest subset sums of a non-negative array (by sorted order) without generating all 2^n subsets. It avoids duplicates in practice and is safe because k is small. The transformation subsequenceSum = base - subsetSum is valid because each subsequence differs from the base by excluding some absolute values.

---

Java implementation

import java.util.*;

/**
 * Find the k-th largest subsequence sum (k-Sum).
 */
public class KSumOfArray {
    // Node represents a subset-sum value built using a[0..idx] with last included element a[idx]
    private static class Node {
        long sum;
        int idx;
        Node(long s, int i) { sum = s; idx = i; }
    }

    public long kSum(int[] nums, int k) {
        int n = nums.length;
        long base = 0L;
        // build absolute values and compute base
        long[] a = new long[n];
        int m = 0;
        for (int x : nums) {
            if (x > 0) base += x;            // take positive to build base
            a[m++] = Math.abs((long)x);
        }

        // sort absolute values ascending
        Arrays.sort(a, 0, m);

        // If k == 1, the answer is base (subset-sum 0)
        if (k == 1) return base;

        // Min-heap by sum
        PriorityQueue pq = new PriorityQueue<>(Comparator.comparingLong(node -> node.sum));
        // initialize with first non-zero subset sum a[0] if exists
        if (m > 0) pq.offer(new Node(a[0], 0));

        long currentSubsetSum = 0L; // this will hold the popped subset-sum
        int count = 1; // we've "taken" the zero subset-sum already (t=0 corresponds to base)

        while (!pq.isEmpty() && count < k) {
            Node cur = pq.poll();
            currentSubsetSum = cur.sum;
            int i = cur.idx;
            // generate two children using next index i+1
            if (i + 1 < m) {
                // extend current subset by taking a[i+1]
                pq.offer(new Node(currentSubsetSum + a[i + 1], i + 1));
                // replace a[i] by a[i+1] (drop last included, include next)
                pq.offer(new Node(currentSubsetSum - a[i] + a[i + 1], i + 1));
            }
            count++;
        }

        // If we enumerated k subset-sums, the k-th smallest subset-sum is:
        // - if k == 1 => 0 (handled earlier)
        // - else => currentSubsetSum (value popped at count==k)
        // The k-th largest subsequence sum = base - currentSubsetSum
        return base - currentSubsetSum;
    }

    // Quick test harness
    public static void main(String[] args) {
        KSumOfArray solver = new KSumOfArray();

        System.out.println(solver.kSum(new int[]{2,4,-2}, 5)); // expect 2
        System.out.println(solver.kSum(new int[]{1,-2,3,4,-10,12}, 16)); // expect 10
    }
}

---

Final remarks / caveats

  • Use long for all sums to avoid overflow (nums[i] up to 1e9 and sums up to 1e14).
  • The technique relies on k being small. If k were large (comparable to 2^n), we'd need a different approach or it would be infeasible.
  • This implementation assumes we count the empty subset-sum 0 as the first (k=1 ‚Üí largest subsequence sum = base). That matches the problem definition examples.
