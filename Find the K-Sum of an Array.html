Find the K-Sum of an Array

Medium

Find the K-Sum of an Array Example 1:

Input: nums = [2,4,-2], k = 5 Output: 2 Explanation: All the possible subsequence sums that we can obtain are the following sorted in decreasing order: 6, 4, 4, 2, 2, 0, 0, -2. The 5-Sum of the array is 2. Example 2:

Input: nums = [1,-2,3,4,-10,12], k = 16 Output: 10 Explanation: The 16-Sum of the array is 10.

Constraints:

n == nums.length 1 <= n <= 10^5 -10^9 <= nums[i] <= 10^9 1 <= k <= min(2000, 2^n)

Key idea (short)

  1. Let base = sum(max(0, nums[i])) ‚Äî the maximum subsequence sum (take all positives).
  2. Every subsequence sum can be written as base - t, where t is the sum of absolute values of elements we _remove_ from that maximal choice (i.e., choosing not to take a positive or taking a negative). Let a[] be the absolute values |nums[i]| sorted ascending.
  3. We need the k-th largest subsequence sum ‚Üí equivalently the k-th smallest t. The smallest t is 0 (empty removal), so the k-th smallest t can be enumerated using a min-heap technique that generates the smallest subset sums of a[] in increasing order (we only need up to k sums and k ‚â§ 2000).
  4. Return base - t_k.

This avoids enumerating all 2^n subsets because we only generate the first k smallest subset sums via a heap-based BFS over combinations (classic technique used when k is small).

---

Correct & efficient Java implementation

import java.util.*;

public class Solution {
    private static class Node {
        long sum; // subset-sum value
        int idx;  // last included index in a[]
        Node(long s, int i) { sum = s; idx = i; }
    }

    public long kSum(int[] nums, int k) {
        int n = nums.length;

        // 1) compute base: sum of positive numbers
        long base = 0L;
        for (int x : nums) if (x > 0) base += x;

        // If we want the largest subsequence sum, k==1 => it's base
        if (k == 1) return base;

        // 2) build array of absolute values and sort ascending
        long[] a = new long[n];
        for (int i = 0; i < n; i++) a[i] = Math.abs((long) nums[i]);
        Arrays.sort(a);

        // 3) min-heap to enumerate smallest subset sums of a[]
        PriorityQueue pq = new PriorityQueue<>(Comparator.comparingLong(node -> node.sum));
        // push first non-zero subset-sum candidate (a[0]) if there is any element
        if (n > 0) pq.offer(new Node(a[0], 0));

        long currentSubsetSum = 0L; // the last popped subset-sum (0 corresponds to empty subset)
        int count = 1; // we've already "used" the empty subset-sum = 0 as the 1st smallest

        while (!pq.isEmpty() && count < k) {
            Node cur = pq.poll();
            currentSubsetSum = cur.sum;
            int i = cur.idx;
            if (i + 1 < n) {
                // extend current subset by a[i+1]
                pq.offer(new Node(currentSubsetSum + a[i + 1], i + 1));
                // replace last included element a[i] by a[i+1] (drop a[i], include a[i+1])
                pq.offer(new Node(currentSubsetSum - a[i] + a[i + 1], i + 1));
            }
            count++;
        }

        // The k-th smallest subset-sum t_k is:
        // - if k==1: t_k = 0 (handled earlier)
        // - else: t_k = currentSubsetSum (the value popped at count==k)
        // So the k-th largest subsequence sum = base - t_k
        return base - currentSubsetSum;
    }


}

---

Complexity

  • Sorting a[]: O(n log n).
  • Heap enumerations: at most O(k) pops and O(k) pushes; each heap operation O(log k). So O(k log k) for the enumeration (with k ‚â§ 2000).
  • Overall: O(n log n + k log k) time, O(n + k) space.

---

Notes & robustness

  • We use long for sums to avoid overflow (nums[i] up to 1e9, n up to 1e5).
  • The algorithm counts the empty subset (sum 0) as the first (k=1 ‚Üí return base), matching the problem examples.
  • The heap-generation technique can produce duplicate subset sums (which is OK ‚Äî duplicates correspond to different subsets and should all be counted).
  • By constraints k <= min(2000, 2^n), the heap will be able to generate k distinct enumerations ‚Äî we safely assume we won't prematurely exhaust combinations.
