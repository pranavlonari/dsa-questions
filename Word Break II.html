Word Break II

Medium DFSGraphHash TableArrayString

Problem restatement

Given a string s, return all possible sentences formed by concatenating words from wordDict (words can be reused) such that the concatenation (without spaces) equals s. Return an empty list if none exist.

Examples

  • Example 1
Input: s = "catsanddog", wordDict = ["cat","cats","and","sand","dog"] Output: ["cats and dog","cat sand dog"]
  • Example 2
Input: s = "pineapplepenapple", wordDict = ["apple","pen","applepen","pine","pineapple"] Output: ["pine apple pen apple","pineapple pen apple","pine applepen apple"]
  • Example 3
Input: s = "catsandog", wordDict = ["cats","dog","sand","and","cat"] Output: []

Constraints (provided): 1 <= s.length <= 20 1 <= wordDict.length <= 1000 1 <= wordDict[i].length <= 10 s and wordDict[i] are lowercase, wordDict strings unique. Input guarantees total output length ‚â§ 10^5.

Edge cases to call out:

  • s empty (but by constraints length‚â•1) ‚Äî treat as base.
  • No decomposition possible -> return [].
  • Many overlapping word choices -> exponential number of sentences (but bounded here).

---

Quick script to say to the interviewer

  1. "I will first check feasibility for suffixes using a DP boolean (classic Word Break) to prune impossible branches."
  2. "Then I'll do DFS + memoization: from index i build all sentences for s[i:] by trying every word that matches at i, combining current word + sentences from suffix."
  3. "This yields correct results and avoids recomputation via memo. Time is exponential in the worst-case but acceptable given constraints; DP prunes a lot."
  4. "I'll implement the DP + DFS memo approach in Java now."

---

Approaches considered (pros/cons)

  1. Brute-force DFS without memo ‚Äî try all splits; correct but repeats subproblems; can be exponential and very slow.
  2. DFS + memoization (chosen) ‚Äî memoize results for each start index to avoid recomputation. Time still worst-case exponential (number of valid sentences) but much faster in practice.
  3. DP building all sentences bottom-up ‚Äî can be done but more complex; same idea as memoized DFS.
  4. Trie + DFS ‚Äî useful if wordDict is large and you want faster prefix checks; not necessary here because word lengths ‚â§ 10 and constraints small.

I choose DP (feasibility) + DFS with memo: simple, robust, and interviewer-friendly.

---

Detailed chosen approach (step-by-step)

Data structures:

  • Set dict ‚Äî O(1) membership checks.
  • boolean[] canBreak ‚Äî DP feasibility: canBreak[i] true if s[i:] can be segmented.
  • Map> memo ‚Äî memoization mapping start index -> list of sentences for s.substring(start).

Steps:

  1. Build dict from wordDict and compute canBreak boolean DP from back to front:
  • canBreak[n] = true (empty suffix).
  • For i from n-1 down to 0: for each word length len ‚â§ 10 (or each word in dict if you prefer), if s.substring(i,i+len).equals(word) and canBreak[i+len] is true then canBreak[i] = true.
  • This lets us prune branches that cannot lead to a full segmentation.
  1. If canBreak[0] is false, return [] immediately.
  2. DFS with memo: dfs(start) returns List for s.substring(start).
  • If start == n return [""] (one empty sentence, used for concatenation).
  • If memo contains start, return it.
  • For each word in dict (or loop lengths and substring) that matches s at start:
  • If canBreak[start + word.length()] is false continue (prune).
  • Recurse restList = dfs(start + word.length()).
  • For each rest in restList:
  • If rest is empty, add word else add word + " " + rest.
  • Store results in memo and return them.
  1. Return dfs(0).

Why it works: canBreak ensures we only explore viable suffixes; memo avoids recomputing solutions for the same start index. Building strings by combining current word with all sentences for suffix enumerates all valid partitions.

---

Complexity (discussion)

  • Let n = s.length() (‚â§ 20).
  • Let W be number of dictionary words (‚â§ 1000) and Lmax ‚â§ 10 word length.
  • Time (informal): Worst-case number of valid sentences can be exponential in n (e.g., many short words). Building each sentence requires concatenations. A common upper bound often given is O(n _ (# of sentences) _ average concat cost) or O(n \* 2^n) in pathological cases. However with n ‚â§ 20 and output bounded by 1e5, this is fine. The DP + memo pruning reduces wasted work considerably.
  • Space: Memo stores each start index's result; total space bounded by the total size of all returned sentences (‚â§ 10^5) plus O(n) for DP and recursion stack. So overall O(output size + n).

Note for interview: state that exact tight runtime is output-sensitive; with small n and constraints this method is standard and accepted.

Java implementation (DP + DFS + memo) ‚Äî production ready

import java.util.*;

public class WordBreakII {
    public List wordBreak(String s, List wordDict) {
        int n = s.length();
        Set dict = new HashSet<>(wordDict);

        // 1) DP feasibility: canBreak[i] == true if s[i:] can be segmented
        boolean[] canBreak = new boolean[n + 1];
        canBreak[n] = true; // empty suffix
        // Precompute possible word lengths to speed up (optional)
        Set wordLens = new HashSet<>();
        for (String w : dict) wordLens.add(w.length());

        for (int i = n - 1; i >= 0; i--) {
            for (int len : wordLens) {
                int j = i + len;
                if (j <= n && canBreak[j]) {
                    if (s.regionMatches(i, s, i, 0)) { /* no-op to silence unused */ }
                    // check substring equality
                    if (s.substring(i, j).length() == len && dict.contains(s.substring(i, j))) {
                        canBreak[i] = true;
                        break;
                    }
                }
            }
        }

        if (!canBreak[0]) return Collections.emptyList();

        // 2) DFS + memo to build all sentences
        Map> memo = new HashMap<>();
        return dfs(0, s, dict, wordLens, canBreak, memo);
    }

    private List dfs(int start,
                             String s,
                             Set dict,
                             Set wordLens,
                             boolean[] canBreak,
                             Map> memo) {
        int n = s.length();
        if (start == n) return Arrays.asList(""); // one empty sentence to concatenate

        if (memo.containsKey(start)) return memo.get(start);

        List res = new ArrayList<>();
        for (int len : wordLens) {
            int end = start + len;
            if (end > n) continue;
            if (!canBreak[end]) continue; // prune impossible suffix
            String word = s.substring(start, end);
            if (!dict.contains(word)) continue;

            List suffixSentences = dfs(end, s, dict, wordLens, canBreak, memo);
            for (String suffix : suffixSentences) {
                if (suffix.isEmpty()) res.add(word);
                else res.add(word + " " + suffix);
            }
        }

        memo.put(start, res);
        return res;
    }

    // Quick test harness
    public static void main(String[] args) {
        WordBreakII solver = new WordBreakII();

        String s1 = "catsanddog";
        List dict1 = Arrays.asList("cat","cats","and","sand","dog");
        System.out.println(solver.wordBreak(s1, dict1));
        // expected: ["cats and dog","cat sand dog"] (order may vary)

        String s2 = "pineapplepenapple";
        List dict2 = Arrays.asList("apple","pen","applepen","pine","pineapple");
        System.out.println(solver.wordBreak(s2, dict2));
        // expected: ["pine apple pen apple","pineapple pen apple","pine applepen apple"]

        String s3 = "catsandog";
        List dict3 = Arrays.asList("cats","dog","sand","and","cat");
        System.out.println(solver.wordBreak(s3, dict3));
        // expected: []
    }
}

Note about the implementation:
  • I used wordLens to avoid checking every dictionary word at every index ‚Äî we only try substring lengths that exist in the dict (micro-optimization).
  • canBreak prunes impossible branches early.
  • memo caches List results for each start index to avoid recomputation.

Possible improvements & tradeoffs

  • Use Trie if wordDict is very large to speed prefix checks (though here word length ‚â§ 10 so substring checks are fine).
  • Build bottom-up DP that accumulates sentences at indices ‚Äî similar but DFS + memo is clearer in interview.
  • If memory is tight and n larger, you might stream results, but Word Break II expects full list.
