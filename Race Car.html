Race car

Your car starts at position 0 and speed +1 on an infinite number line. Your car can go into negative positions. Your car drives automatically according to a sequence of instructions 'A' (accelerate) and 'R' (reverse):

When you get an instruction 'A', your car does the following: position += speed speed \*= 2 When you get an instruction 'R', your car does the following: If your speed is positive then speed = -1 otherwise speed = 1 Your position stays the same. For example, after commands "AAR", your car goes to positions 0 --> 1 --> 3 --> 3, and your speed goes to 1 --> 2 --> 4 --> -1.

Given a target position target, return the length of the shortest sequence of instructions to get there.

Example 1:

Input: target = 3 Output: 2 Explanation: The shortest instruction sequence is "AA". Your position goes from 0 --> 1 --> 3. Example 2:

Input: target = 6 Output: 5 Explanation: The shortest instruction sequence is "AAARA". Your position goes from 0 --> 1 --> 3 --> 7 --> 7 --> 6.

Constraints:

1 <= target <= 10^4

---

Problem summary (short)

Start at position 0 with speed +1. Commands:

  • A (accelerate): pos += speed; speed *= 2
  • R (reverse): if (speed > 0) speed = -1; else speed = 1 (position unchanged)

Given target (1 ‚â§ target ‚â§ 10‚Å¥), find the minimum length (number of commands) to reach target.

---

Key idea / intuition

If you press A repeatedly n times starting from the initial state you will exactly reach position 2^n - 1 and have speed 2^n. Those special positions partition the line of reachable positions in powers-of-two blocks.

For a given target there are two useful patterns:

1) Reach exactly 2^n - 1 (perfect)

If target == 2^n - 1 for some n, the optimal sequence is simply n As. So answer = n.

2) Overshoot and reverse

If you do n As (reach 2^n - 1) and that overshoots target (i.e. 2^n - 1 > target), one option is:

  • Do n As, then R (now speed flips), then solve the remaining distance: you need to go back by 2^n - 1 - target. That remaining subproblem is the same kind of problem but smaller. Total commands: n (A's) + 1 (R) + dp[2^n - 1 - target].

3) Undershoot, reverse early, and approach

A better strategy sometimes is: do n-1 As to go to 2^{n-1}-1 (still less than target), then R to reverse, then perform m As (moving backward some amount), then R again to fix direction, then finish the remaining small forward distance. Formally:

  • Do n-1 A, R, then m A, R, then handle the remaining distance rem = target - ( (2^{n-1}-1) - (2^m -1) ) via dp[rem].
  • The cost for this plan is (n-1) (first A's) + 1 (R) + m (A's) + 1 (R) + dp[rem].

We try all m from 0 to n-2 (so that the second A burst is shorter than the first) and take the minimum.

Why these are the only cases to consider? Because the optimal sequence either reaches some 2^n-1 exactly at some time (and possibly reverses there) or it reaches 2^{n-1}-1 (the largest position less than target) and then uses the two-reverse pattern described. Using these building blocks and recursion/memoization yields the optimum.

---

Dynamic programming approach (bottom-up)

Let dp[t] be the minimum number of commands to reach position t. We compute dp[1..target]:

  • Let n = ceil(log2(t+1)) (i.e. smallest n such that 2^n - 1 >= t).
  • If 2^n - 1 == t, dp[t] = n.
  • Otherwise: initialize dp[t] = n + 1 + dp[(2^n - 1) - t] (overshoot case).
  • For all m from 0 to n-2:
  • forward = 2^{n-1} - 1
  • back = 2^{m} - 1
  • rem = t - (forward - back) (this is non-negative in the considered cases)
  • candidate = (n-1) + 1 + m + 1 + dp[rem]
  • (n-1) A‚Äôs, 1 R, m A‚Äôs, 1 R, then dp[rem]
  • take dp[t] = min(dp[t], candidate)

We fill dp from 1 up to target ‚Äî dp uses only smaller values (rem < t) so bottom-up works.

---

Complexity

  • For each t we compute n = O(log t) and run O(n) inner loop. Summed across t ‚â§ T gives about O(T log T) time (with small constants). For T ‚â§ 10^4 this is easily fine.
  • Space: O(T) for the dp array.

---

Correctness sketch

The recurrence enumerates all useful patterns by considering the nearest power-of-two boundaries. The two pattern families (overshoot and undershoot with a second reverse) are enough to cover any optimal instruction sequence because A operations double speed, so optimal strategies always align with these blocks of 2^k - 1. This is a standard and well-known correct DP for this problem.

---

Java implementation (bottom-up DP)

import java.util.Arrays;

class Solution {
    public int racecar(int target) {
        // dp[t] = min instructions to reach t
        int[] dp = new int[target + 1];
        Arrays.fill(dp, Integer.MAX_VALUE / 2);
        dp[0] = 0;

        for (int t = 1; t <= target; t++) {
            // n = smallest number such that 2^n - 1 >= t
            int n = 32 - Integer.numberOfLeadingZeros(t); // n = ceil(log2(t+1))
            int fullPos = (1 << n) - 1;
            if (fullPos == t) {
                dp[t] = n; // n 'A's
                continue;
            }

            // Option 1: go past target to fullPos, reverse, and come back
            dp[t] = n + 1 + dp[fullPos - t];

            // Option 2: do (n-1) 'A's, reverse, do m 'A's (in other direction), reverse again, then finish
            // try all m from 0 to n-2
            int forward = (1 << (n - 1)) - 1; // position after n-1 As
            for (int m = 0; m <= n - 2; m++) {
                int back = (1 << m) - 1; // distance covered by m As
                int rem = t - (forward - back);
                if (rem >= 0) {
                    // (n-1) A's + R + m A's + R + dp[rem]
                    int candidate = (n - 1) + 1 + m + 1 + dp[rem];
                    dp[t] = Math.min(dp[t], candidate);
                }
            }
        }
        return dp[target];
    }


}

---

Notes & practical tips

  • Integer.numberOfLeadingZeros is used to compute n = ceil(log2(t+1)) reliably.
  • dp[t] references only smaller indices: fullPos - t < t and rem < t, so bottom-up fill is safe.
  • The DP approach is fast enough up to target = 10^4. There exists also a BFS solution over (position,speed) state space, but that can be heavier and more memory-hungry.
  • You can also implement the recurrence with recursion + memoization; bottom-up is straightforward and avoids recursion limits.
