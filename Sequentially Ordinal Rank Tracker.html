Sequentially Ordinal Rank Tracker

A scenic location is represented by its name and attractiveness score, where name is a unique string among all locations and score is an integer. Locations can be ranked from the best to the worst. The higher the score, the better the location. If the scores of two locations are equal, then the location with the lexicographically smaller name is better.

You are building a system that tracks the ranking of locations with the system initially starting with no locations. It supports:

Adding scenic locations, one at a time. Querying the ith best location of all locations already added, where i is the number of times the system has been queried (including the current query). For example, when the system is queried for the 4th time, it returns the 4th best location of all locations already added. Note that the test data are generated so that at any time, the number of queries does not exceed the number of locations added to the system.

Implement the SORTracker class:

SORTracker() Initializes the tracker system. void add(string name, int score) Adds a scenic location with name and score to the system. string get() Queries and returns the ith best location, where i is the number of times this method has been invoked (including this invocation).

Example 1:

Input ["SORTracker", "add", "add", "get", "add", "get", "add", "get", "add", "get", "add", "get", "get"] [[], ["bradford", 2], ["branford", 3], [], ["alps", 2], [], ["orland", 2], [], ["orlando", 3], [], ["alpine", 2], [], []] Output [null, null, null, "branford", null, "alps", null, "bradford", null, "bradford", null, "bradford", "orland"]

Explanation SORTracker tracker = new SORTracker(); // Initialize the tracker system. tracker.add("bradford", 2); // Add location with name="bradford" and score=2 to the system. tracker.add("branford", 3); // Add location with name="branford" and score=3 to the system. tracker.get(); // The sorted locations, from best to worst, are: branford, bradford. // Note that branford precedes bradford due to its higher score (3 > 2). // This is the 1st time get() is called, so return the best location: "branford". tracker.add("alps", 2); // Add location with name="alps" and score=2 to the system. tracker.get(); // Sorted locations: branford, alps, bradford. // Note that alps precedes bradford even though they have the same score (2). // This is because "alps" is lexicographically smaller than "bradford". // Return the 2nd best location "alps", as it is the 2nd time get() is called. tracker.add("orland", 2); // Add location with name="orland" and score=2 to the system. tracker.get(); // Sorted locations: branford, alps, bradford, orland. // Return "bradford", as it is the 3rd time get() is called. tracker.add("orlando", 3); // Add location with name="orlando" and score=3 to the system. tracker.get(); // Sorted locations: branford, orlando, alps, bradford, orland. // Return "bradford". tracker.add("alpine", 2); // Add location with name="alpine" and score=2 to the system. tracker.get(); // Sorted locations: branford, orlando, alpine, alps, bradford, orland. // Return "bradford". tracker.get(); // Sorted locations: branford, orlando, alpine, alps, bradford, orland. // Return "orland".

Constraints:

name consists of lowercase English letters, and is unique among all locations. 1 <= name.length <= 10 1 <= score <= 10^5 At any time, the number of calls to get does not exceed the number of calls to add. At most 4 \* 10^4 calls in total will be made to add and get.

this is the classic SORTracker problem. .

---

Short script to the interviewer (copy/paste)

‚ÄúI maintain two heaps: a min-heap right of all not-yet-selected locations ordered best ‚Üí worst and a max-heap left of already-selected locations ordered worst ‚Üí best (i.e. left.peek() is the current answer).

  • add(name,score): insert into right, then if right has an element better than left.peek() swap the two so left always contains the q best elements (q = number of get() calls so far).
  • get(): move the current best from right into left (increasing q by 1) and return left.peek().name.
This yields O(log n) per operation and keeps the invariant that left stores the top-q locations and left.peek() is the q-th best (the result to return).‚Äù

---

Intuition & correctness

  • Use a consistent ranking comparator: higher score is better; on ties the lexicographically smaller name is better.
  • right is a min-heap by that comparator so right.peek() is the overall best among not-selected.
  • left is a _reverse_ comparator (max-heap in ranking order) so left.peek() is the worst among selected ‚Üí that's exactly the q-th best.
  • On add, we push into right. If right.peek() is better than left.peek() (i.e. a newly added item outranks someone already selected), we swap to keep left containing the best q.
  • On get, we increase q by moving right.peek() ‚Üí left. Then left.peek() is the q-th best. The problem guarantee ensures right is non-empty when get is called.

---

Complexity

  • add: O(log n) for heap insert (plus at most one swap of two O(log n) ops).
  • get: O(log n) for heap poll/offer.
  • Space: O(n) total.
  • Works within constraints (‚â§ 4¬∑10‚Å¥ calls).

Java implementation

import java.util.*;

public class SORTracker {
    // Location class
    private static class Loc {
        String name;
        int score;
        Loc(String name, int score) { this.name = name; this.score = score; }
    }

    // comparator: returns negative when a is BETTER than b
    private static final Comparator bestComparator = (a, b) -> {
        if (a.score != b.score) return Integer.compare(b.score, a.score); // higher score first
        return a.name.compareTo(b.name); // lexicographically smaller name first
    };

    // left comparator: reverse of bestComparator (so left.peek() is worst among selected)
    private static final Comparator worstAmongSelectedComparator = (a, b) -> {
        // Reverse the ordering of bestComparator
        if (a.score != b.score) return Integer.compare(a.score, b.score); // lower score first
        return b.name.compareTo(a.name); // lexicographically larger name first
    };

    private final PriorityQueue left;   // stores selected top-q (worst among them at head)
    private final PriorityQueue right;  // stores remaining (best among them at head)

    public SORTracker() {
        left = new PriorityQueue<>(worstAmongSelectedComparator);
        right = new PriorityQueue<>(bestComparator);
    }

    public void add(String name, int score) {
        Loc loc = new Loc(name, score);
        // Always add to right first
        right.offer(loc);
        // If left is non-empty and right's best is better than left's worst, swap them.
        if (!left.isEmpty() && !right.isEmpty() && bestComparator.compare(right.peek(), left.peek()) < 0) {
            Loc r = right.poll();
            Loc l = left.poll();
            left.offer(r);
            right.offer(l);
        }
    }

    public String get() {
        // Move the current best from right to left (increase query-count)
        if (!right.isEmpty()) {
            left.offer(right.poll());
        }
        // left.peek() is the current answer
        return left.peek().name;
    }

    // --- Quick example run using the sample from prompt ---
    public static void main(String[] args) {
        SORTracker tracker = new SORTracker();
        tracker.add("bradford", 2);
        tracker.add("branford", 3);
        System.out.println(tracker.get()); // "branford" (1st get)
        tracker.add("alps", 2);
        System.out.println(tracker.get()); // "alps" (2nd get)
        tracker.add("orland", 2);
        System.out.println(tracker.get()); // "bradford" (3rd get)
        tracker.add("orlando", 3);
        System.out.println(tracker.get()); // "bradford" (4th get)
        tracker.add("alpine", 2);
        System.out.println(tracker.get()); // "bradford" (5th get)
        System.out.println(tracker.get()); // "orland"  (6th get)
    }
}

Final talking points (to say after coding)

  • Explain the two-heap invariant: left contains top-q locations, left.peek() is the q-th best.
  • Describe the comparators (score desc, name asc), and why right uses that ordering while left uses the reverse.
  • Complexity: each operation O(log n).
  • Mention that Java TreeSet / TreeMap could also be used, but two heaps are simpler and efficient.
