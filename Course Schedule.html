Course Schedule

There are a total of numCourses courses you have to take, labeled from 0 to numCourses - 1. You are given an array prerequisites where prerequisites[i] = [ai, bi] indicates that you must take course bi first if you want to take course ai.

For example, the pair [0, 1], indicates that to take course 0 you have to first take course 1. Return true if you can finish all courses. Otherwise, return false.

Example 1:

Input: numCourses = 2, prerequisites = [[1,0]] Output: true Explanation: There are a total of 2 courses to take. To take course 1 you should have finished course 0. So it is possible. Example 2:

Input: numCourses = 2, prerequisites = [[1,0],[0,1]] Output: false Explanation: There are a total of 2 courses to take. To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.

Constraints:

1 <= numCourses <= 2000 0 <= prerequisites.length <= 5000 prerequisites[i].length == 2 0 <= ai, bi < numCourses All the pairs prerequisites[i] are unique.

Short script to say to the interviewer

‚ÄúModel courses as a directed graph where an edge b -> a means b is a prerequisite of a. We can finish all courses iff the directed graph is acyclic. I‚Äôll use Kahn‚Äôs algorithm (BFS topological sort): compute in-degrees, repeatedly remove nodes with in-degree 0 and decrement neighbors ‚Äî if we remove all nodes the graph is acyclic and we can finish; otherwise there‚Äôs a cycle and we cannot. This runs in O(V+E) time and O(V+E) space.‚Äù

---

Approaches considered (brief)

  1. Kahn‚Äôs algorithm (BFS, chosen) ‚Äî maintain in-degree array, queue of zero in-degree nodes. Simple to reason about and easy to implement.
  2. DFS cycle detection ‚Äî run DFS with a visited + recursion-stack marker; detect back edge ‚Üí cycle. Also O(V+E) but slightly more error-prone in interviews for iterative explanation.

I‚Äôll provide the Kahn BFS implementation (clean and standard). I‚Äôll also summarize the DFS idea at the end.

---

Why Kahn‚Äôs algorithm works (intuition)

If there is a node with in-degree 0, it has no prerequisites and can be taken now. Removing it may expose other nodes whose prerequisites are now satisfied. If we can repeatedly remove nodes until none remain, there was no cycle. If any node remains with positive in-degree and the queue is empty, those nodes form (or depend on) a cycle.

---

Complexity

  • Time: O(numCourses + P) where P = prerequisites.length (each edge and node processed constant times).
  • Space: O(numCourses + P) for adjacency lists and in-degree array.

---

Java implementation (Kahn's BFS / topological-sort)

import java.util.*;

public class CourseSchedule {
    // Return true if we can finish all courses
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        // adjacency list: graph[from] -> list of to
        List> graph = new ArrayList<>(numCourses);
        for (int i = 0; i < numCourses; i++) graph.add(new ArrayList<>());

        int[] indeg = new int[numCourses];

        // build graph and indegree
        for (int[] p : prerequisites) {
            int course = p[0];
            int prereq = p[1];
            graph.get(prereq).add(course); // edge prereq -> course
            indeg[course]++;
        }

        // queue of courses with indegree 0 (no prerequisites)
        ArrayDeque q = new ArrayDeque<>();
        for (int i = 0; i < numCourses; i++) {
            if (indeg[i] == 0) q.offer(i);
        }

        int removed = 0;
        while (!q.isEmpty()) {
            int cur = q.poll();
            removed++;
            for (int nei : graph.get(cur)) {
                indeg[nei]--;
                if (indeg[nei] == 0) q.offer(nei);
            }
        }

        // if removed all nodes, no cycle -> can finish all courses
        return removed == numCourses;
    }

    // quick tests
    public static void main(String[] args) {
        CourseSchedule cs = new CourseSchedule();

        System.out.println(cs.canFinish(2, new int[][]{{1,0}}));              // true
        System.out.println(cs.canFinish(2, new int[][]{{1,0},{0,1}}));      // false
        System.out.println(cs.canFinish(4, new int[][]{{1,0},{2,0},{3,1},{3,2}})); // true
        System.out.println(cs.canFinish(1, new int[][]{}));                 // true
    }
}

---

DFS alternative (brief)

  • Run DFS from every unvisited node. Use three-state array: 0 = unvisited, 1 = visiting (in recursion stack), 2 = visited.
  • If during DFS you try to visit a node with state 1, you found a back edge ‚Üí cycle ‚Üí return false.
  • If no DFS finds a back edge, return true.
  • Also O(V+E) time and O(V) recursion stack in worst case.

---

Edge cases / tests to mention

  • numCourses = 1, prerequisites = [] ‚Üí true.
  • Empty prerequisites ‚Üí always true.
  • Single cycle ‚Üí false (e.g., [[0,1],[1,0]]).
  • Multiple independent components ‚Äî must ensure all nodes processed.
  • Large numCourses with many prerequisites ‚Äî algorithm scales to constraints.
