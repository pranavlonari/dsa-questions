Find Good Days to Rob the Bank

Example 1:

Input: security = [5,3,3,3,5,6,2], time = 2 Output: [2,3] Explanation: On day 2, we have security[0] >= security[1] >= security[2] <= security[3] <= security[4]. On day 3, we have security[1] >= security[2] >= security[3] <= security[4] <= security[5]. No other days satisfy this condition, so days 2 and 3 are the only good days to rob the bank. Example 2:

Input: security = [1,1,1,1,1], time = 0 Output: [0,1,2,3,4] Explanation: Since time equals 0, every day is a good day to rob the bank, so return every day. Example 3:

Input: security = [1,2,3,4,5,6], time = 2 Output: [] Explanation: No day has 2 days before it that have a non-increasing number of guards. Thus, no day is a good day to rob the bank, so return an empty list.

Constraints:

1 <= security.length <= 10^5 0 <= security[i], time <= 10^5

Problem restatement (short)

We are given an array security and an integer time. A day i is good if the time days before i are non-increasing (security[j] >= security[j+1]), and the time days after i are non-decreasing (security[j] <= security[j+1]). Return all good days' indices (in any order).

Quick script to say to the interviewer

  1. ‚ÄúI‚Äôll precompute two arrays: left[i] = number of consecutive non-increasing days ending at i, and right[i] = number of consecutive non-decreasing days starting at i.‚Äù
  2. ‚ÄúThen day i is valid iff left[i] >= time and right[i] >= time. This is O(n) time and O(n) space.‚Äù
  3. ‚ÄúIf time == 0 every day qualifies; otherwise we only check indices time..n-1-time.‚Äù

---

Intuition

For each day i we need to know:

  • how many consecutive previous days (including i) satisfy security[k-1] >= security[k] ‚Äî call this left[i], and
  • how many consecutive following days (including i) satisfy security[k] <= security[k+1] ‚Äî call this right[i].

If both counts are at least time, then i has time days before that are non-increasing and time days after that are non-decreasing.

---

Algorithm (O(n) time, O(n) space)

  1. If time == 0, return all indices 0..n-1.
  2. Build left:
  • left[0] = 0.
  • For i = 1..n-1: if security[i] <= security[i-1] then left[i] = left[i-1] + 1 else left[i] = 0.
  • Build right:
  • right[n-1] = 0.
  • For i = n-2 .. 0: if security[i] <= security[i+1] then right[i] = right[i+1] + 1 else right[i] = 0.
  • Collect all i where left[i] >= time && right[i] >= time.
  • Return the list.

---

Complexity

  • Time: O(n) ‚Äî each array scanned a constant number of times.
  • Space: O(n) ‚Äî two integer arrays left and right of size n. (Can be reduced to O(1) with two passes and window technique, but O(n) is simplest.)

---

Java implementation (clean & commented)

import java.util.*;

public class GoodDaysToRobBank {
    public List goodDaysToRobBank(int[] security, int time) {
        int n = security.length;
        List res = new ArrayList<>();
        if (time == 0) {
            // every day is valid
            for (int i = 0; i < n; i++) res.add(i);
            return res;
        }
        if (n == 0) return res;

        int[] left = new int[n];  // left[i] = consecutive non-increasing days ending at i (excluding comparisons beyond)
        int[] right = new int[n]; // right[i] = consecutive non-decreasing days starting at i

        // build left: compare current with previous
        left[0] = 0;
        for (int i = 1; i < n; i++) {
            if (security[i] <= security[i - 1]) left[i] = left[i - 1] + 1;
            else left[i] = 0;
        }

        // build right: compare current with next
        right[n - 1] = 0;
        for (int i = n - 2; i >= 0; i--) {
            if (security[i] <= security[i + 1]) right[i] = right[i + 1] + 1;
            else right[i] = 0;
        }

        // collect indices that satisfy both sides
        for (int i = 0; i < n; i++) {
            if (left[i] >= time && right[i] >= time) res.add(i);
        }
        return res;
    }

    // quick tests
    public static void main(String[] args) {
        GoodDaysToRobBank solver = new GoodDaysToRobBank();

        System.out.println(solver.goodDaysToRobBank(new int[]{5,3,3,3,5,6,2}, 2)); // [2,3]
        System.out.println(solver.goodDaysToRobBank(new int[]{1,1,1,1,1}, 0));       // [0,1,2,3,4]
        System.out.println(solver.goodDaysToRobBank(new int[]{1,2,3,4,5,6}, 2));   // []
    }
}

---

Edge cases to mention

  • time == 0 ‚Üí all days valid.
  • time larger than possible window (e.g., time > n/2) ‚Üí likely empty result.
  • Constant arrays (all equal) ‚Äî many days may satisfy both conditions.
  • Large n up to 1e5 ‚Äî algorithm still O(n).
