Find Good Days to Rob the Bank

Example 1:

Input: security = [5,3,3,3,5,6,2], time = 2 Output: [2,3] Explanation: On day 2, we have security[0] >= security[1] >= security[2] <= security[3] <= security[4]. On day 3, we have security[1] >= security[2] >= security[3] <= security[4] <= security[5]. No other days satisfy this condition, so days 2 and 3 are the only good days to rob the bank. Example 2:

Input: security = [1,1,1,1,1], time = 0 Output: [0,1,2,3,4] Explanation: Since time equals 0, every day is a good day to rob the bank, so return every day. Example 3:

Input: security = [1,2,3,4,5,6], time = 2 Output: [] Explanation: No day has 2 days before it that have a non-increasing number of guards. Thus, no day is a good day to rob the bank, so return an empty list.

Constraints:

1 <= security.length <= 10^5 0 <= security[i], time <= 10^5

What to say to the interviewer (one-liner)

> ‚ÄúPrecompute for each day how many consecutive non-increasing days end at that index (left[i]) and how many consecutive non-decreasing days start at that index (right[i]). A day i is good iff left[i] >= time and right[i] >= time. This is O(n) time and O(n) space.‚Äù

Why it works (quick intuition)

  • left[i] counts how many days up to i follow security[j] >= security[j+1].
  • right[i] counts how many days from i follow security[j] <= security[j+1].
If both are at least time, then there are time days before i that are non-increasing and time days after i that are non-decreasing.

---

Algorithm steps

  1. If time == 0 return all indices immediately.
  2. Build left[]: iterate left‚Üíright; left[0] = 0; for i=1..n-1 if security[i] <= security[i-1] then left[i] = left[i-1] + 1 else left[i] = 0.
  3. Build right[]: iterate right‚Üíleft; right[n-1] = 0; for i=n-2..0 if security[i] <= security[i+1] then right[i] = right[i+1] + 1 else right[i] = 0.
  4. Collect all i where left[i] >= time && right[i] >= time.

---

Java implementation

import java.util.*;

class Solution {
    public List goodDaysToRobBank(int[] security, int time) {
        int n = security.length;
        List res = new ArrayList<>();
        if (time == 0) {
            for (int i = 0; i < n; i++) res.add(i);
            return res;
        }
        if (n == 0) return res;

        int[] left = new int[n];  // number of consecutive non-increasing days ending at i
        int[] right = new int[n]; // number of consecutive non-decreasing days starting at i

        left[0] = 0;
        for (int i = 1; i < n; i++) {
            if (security[i] <= security[i - 1]) left[i] = left[i - 1] + 1;
            else left[i] = 0;
        }

        right[n - 1] = 0;
        for (int i = n - 2; i >= 0; i--) {
            if (security[i] <= security[i + 1]) right[i] = right[i + 1] + 1;
            else right[i] = 0;
        }

        for (int i = 0; i < n; i++) {
            if (left[i] >= time && right[i] >= time) res.add(i);
        }
        return res;
    }

    // Optional quick test
    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.goodDaysToRobBank(new int[]{5,3,3,3,5,6,2}, 2)); // [2,3]
        System.out.println(sol.goodDaysToRobBank(new int[]{1,1,1,1,1}, 0));       // [0,1,2,3,4]
        System.out.println(sol.goodDaysToRobBank(new int[]{1,2,3,4,5,6}, 2));    // []
    }
}

---

Complexity

  • Time: O(n) ‚Äî two linear passes + one scan for outputs.
  • Space: O(n) for left and right (can be reduced to O(1) with more careful window checks, but O(n) is simple and clear).
