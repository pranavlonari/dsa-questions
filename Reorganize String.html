Reorganize String

Medium
BFSGraphArrayStringGreedy

Problem restatement: Given a string s, rearrange its characters so that no two adjacent characters are the same. Return any valid rearrangement or return "" if it's impossible.

Input s = "aab"

Output "aba"

Input s = "aaab"

Output ""

Constraints (provided): 1 <= s.length <= 500 s consists of lowercase English letters.

---

Problem summary (short)

Rearrange characters of s so that no two adjacent characters are equal. Return any valid rearrangement, or "" if impossible.

Key observation: if any character appears more than (n+1)/2 times (where n = s.length()), it is impossible ‚Äî there aren‚Äôt enough ‚Äúother‚Äù characters to separate its occurrences.

---

Two common approaches (both correct)

1) Greedy with max-heap (priority queue) ‚Äî clear & robust

  • Count frequency of each character.
  • If maxFreq > (n+1)/2 return "".
  • Use a max-heap keyed by remaining frequency.
  • Repeatedly take the two most frequent characters (say a and b), append a then b to the result, decrement their counts and push back if still positive. This ensures we never place identical characters adjacent.
  • If one character remains at the end, it will have frequency 1 (because of the maxFreq check) and can be safely appended.

This is easy to reason about and works in O(n log k) time where k ‚â§ 26.

2) Fill-even-then-odd indices (O(n) time)

  • Place the most frequent character first at indices 0,2,4,... then fill remaining characters into remaining even indices then odd indices.
  • This avoids a heap and is O(n) time and O(26) extra space for counts.
  • Requires the same feasibility check maxFreq <= (n+1)/2.

Both solve the problem; I'll provide the heap solution first (easy to understand), and then show the index-filling method as an alternative.

---

Correctness intuition

  • Heap method always picks two different characters with the highest remaining counts; appending them reduces the chance of having that char again adjacent.
  • The feasibility check maxFreq <= (n+1)/2 is necessary and sufficient:
  • If maxFreq > (n+1)/2, the most frequent char must appear adjacent somewhere (pigeonhole).
  • If maxFreq <= (n+1)/2, the greedy algorithms above can always place characters to avoid adjacency.

---

Complexity

Let n = s.length(), alphabet size k ‚â§ 26.

  • Counting frequencies: O(n).
  • Heap solution: each push/pop is O(log k) and we do O(n) operations ‚Üí O(n log k) which is effectively O(n).
  • Index-filling solution: O(n) time only.
  • Space: O(k) extra (counts, heap elements) and O(n) for the result string.

Detailed Java implementation (max-heap approach)

import java.util.*;

class Solution {
    public String reorganizeString(String s) {
        int n = s.length();
        // frequency of each lowercase letter
        int[] freq = new int[26];
        for (char c : s.toCharArray()) freq[c - 'a']++;

        // find the most frequent character
        int maxFreq = 0;
        for (int f : freq) maxFreq = Math.max(maxFreq, f);
        if (maxFreq > (n + 1) / 2) return ""; // impossible

        // max-heap: entries sorted by remaining count descending
        PriorityQueue pq = new PriorityQueue<>((a, b) -> b[1] - a[1]);
        for (int i = 0; i < 26; i++) {
            if (freq[i] > 0) pq.offer(new int[]{i, freq[i]}); // [charIndex, count]
        }

        StringBuilder sb = new StringBuilder();
        while (pq.size() >= 2) {
            int[] e1 = pq.poll();
            int[] e2 = pq.poll();
            // append the two different characters
            sb.append((char) ('a' + e1[0]));
            sb.append((char) ('a' + e2[0]));
            // decrement counts and push back if still positive
            if (--e1[1] > 0) pq.offer(e1);
            if (--e2[1] > 0) pq.offer(e2);
        }
        // if one character left, append it (count must be 1)
        if (!pq.isEmpty()) {
            sb.append((char) ('a' + pq.poll()[0]));
        }
        return sb.toString();
    }

    // Optional quick test
    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.reorganizeString("aab"));   // "aba" (or some valid)
        System.out.println(sol.reorganizeString("aaab"));  // ""
        System.out.println(sol.reorganizeString("vvvlo")); // e.g. "vlvov" or similar
    }
}

Alternative (even-odd indices) ‚Äî O(n) ‚Äî concise

class Solution2 {
    public String reorganizeString(String s) {
        int n = s.length();
        int[] cnt = new int[26];
        for (char c : s.toCharArray()) cnt[c - 'a']++;
        int maxIdx = 0;
        for (int i = 1; i < 26; i++) {
            if (cnt[i] > cnt[maxIdx]) maxIdx = i;
        }
        // impossible check
        if (cnt[maxIdx] > (n + 1) / 2) return "";

        char[] res = new char[n];
        int idx = 0;
        // place the most frequent char first at even positions
        while (cnt[maxIdx] > 0) {
            res[idx] = (char) ('a' + maxIdx);
            idx += 2;
            cnt[maxIdx]--;
        }
        // fill remaining characters
        for (int i = 0; i < 26; i++) {
            while (cnt[i] > 0) {
                if (idx >= n) idx = 1; // switch to odd indices
                res[idx] = (char) ('a' + i);
                idx += 2;
                cnt[i]--;
            }
        }
        return new String(res);
    }
}

This method is very efficient and elegant: place the most frequent char spaced out first, then fill remaining slots.

---

Edge cases & tests

  • s.length() == 1 ‚Üí return s.
  • All same characters and n > 1 ‚Üí impossible.
  • Two characters alternating already (e.g. "abab") ‚Üí returns a valid rearrangement (may be same).
  • Random tests should validate no two adjacent chars equal and if result is "" it is indeed impossible.
