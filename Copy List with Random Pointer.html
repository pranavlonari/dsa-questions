Copy List with Random Pointer

Medium GraphLinked ListHash TableTwo Pointers

Problem statement: Given the head of a linked list where each node has an extra random pointer that can point to any node in the list or null, return a deep copy of the list. The deep copy should duplicate both next and random structure (no node from the new list should be the same object as any node in the original list).

Input head = [[7,null],[13,0],[11,4],[10,2],[1,0]] (This representation means nodes with values 7,13,11,10,1; random points to index or null per pair.)

Output [[7,null],[13,0],[11,4],[10,2],[1,0]] (A new list structurally identical to input; nodes are fresh copies.)

Constraints (provided): 0 <= n <= 1000 -10^4 <= Node.val <= 10^4 Node.random is null or points to some node in the linked list.

Example / Edge cases to mention:

  • Empty list (head == null) -> return null.
  • random pointers all null.
  • random pointers pointing to self or earlier/later nodes.
  • Single node with random pointing to self.

Approaches considered (brief pros/cons):

  1. HashMap (value: original node -> copy node) ‚Äî single pass to create copies into map, second pass to wire next and random.
  • Pros: Simple, easy to reason.
  • Cons: O(n) extra space for the map.
  1. Interleaving (chosen) ‚Äî weave copied nodes into original list, set random for copies using originals, then unweave lists.
  • Pros: O(n) time and O(1) extra space (excluding returned nodes).
  • Cons: Slightly more implementation steps but standard and optimal.

I‚Äôll choose the Interleaving approach because it achieves linear time with constant extra space, which is ideal for the given constraints.

Detailed chosen approach ‚Äî Interleaving (step-by-step):

  • Data structures: Use the existing list nodes; create new nodes and insert them immediately after their originals.
  • Steps:
  1. Weave copies: For each original node cur, create copy = new Node(cur.val) and insert it after cur (cur.next = copy; copy.next = nextOriginal).
  • After this pass: original and copy nodes alternate: orig1 -> copy1 -> orig2 -> copy2 -> ....
  1. Assign random pointers for copies: For each original cur, if cur.random != null then cur.next.random = cur.random.next (since cur.next is the copy and cur.random.next is the copy of the random target).
  2. Unweave to restore original and extract copy list: Separate the interleaved list into two lists: restore orig.next = orig.next.next and set copy.next = copy.next.next appropriately.
  • Correctness intuition: Each original node's copy is directly adjacent so it's easy to find corresponding random copy using orig.random.next. Unweaving restores originals while producing the deep-copied list.

Complexity:

  • Time: O(n) ‚Äî three linear passes over the list (weave, assign randoms, unweave). Each node touched constant times.
  • Space: O(1) extra (not counting the output list). Only a few pointers used. (HashMap approach would use O(n) extra.)

Java implementation (clean & commented) ‚Äî Interleaving method:

// Definition for a Node.
class Node {
    int val;
    Node next;
    Node random;
    Node(int val) { this.val = val; this.next = null; this.random = null; }
}

public class CopyListWithRandomPointer {
    public Node copyRandomList(Node head) {
        if (head == null) return null;

        // 1) Weave copied nodes after originals
        Node cur = head;
        while (cur != null) {
            Node next = cur.next;
            Node copy = new Node(cur.val);
            cur.next = copy;
            copy.next = next;
            cur = next;
        }

        // 2) Assign random pointers for copied nodes
        cur = head;
        while (cur != null) {
            Node copy = cur.next;
            if (cur.random != null) {
                // copy.random should point to copy of cur.random which is cur.random.next
                copy.random = cur.random.next;
            } else {
                copy.random = null;
            }
            cur = copy.next; // move to next original node
        }

        // 3) Unweave the lists: restore original list and extract copy list
        cur = head;
        Node pseudoHead = new Node(0); // dummy to form the copied list
        Node copyIter = pseudoHead;
        while (cur != null) {
            Node copy = cur.next;
            Node nextOriginal = copy.next;

            // append copy
            copyIter.next = copy;
            copyIter = copy;

            // restore original
            cur.next = nextOriginal;

            cur = nextOriginal;
        }

        return pseudoHead.next;
    }

    // Optional: HashMap method (straightforward) if interviewer prefers explicit mapping:
    public Node copyRandomListWithMap(Node head) {
        if (head == null) return null;
        Map map = new HashMap<>();
        Node cur = head;
        // create copies
        while (cur != null) {
            map.put(cur, new Node(cur.val));
            cur = cur.next;
        }
        // assign next and random
        cur = head;
        while (cur != null) {
            Node copy = map.get(cur);
            copy.next = (cur.next != null) ? map.get(cur.next) : null;
            copy.random = (cur.random != null) ? map.get(cur.random) : null;
            cur = cur.next;
        }
        return map.get(head);
    }
}

Tests (speak these to the interviewer):

  • head = null ‚Üí null.
  • Single node 1 with random = null ‚Üí copy single node 1 with random = null.
  • Single node 1 with random pointing to itself ‚Üí copied node 1 with random pointing to itself.
  • Provided example: [[7,null],[13,0],[11,4],[10,2],[1,0]] ‚Üí same structure but different object instances.
  • Random pointers forward/backward across list, ensure deep copy (original nodes unchanged).

Possible optimizations / talk-through points:

  • Interleaving approach is already optimal for time and extra space. You can mention tradeoffs: HashMap method is simpler to implement/reason about but uses O(n) space.
  • If node values are large or non-unique, mapping by node references (not values) is necessary ‚Äî both implementations use node identity, not value equality.
  • For concurrency/immutable structures: deep copy semantics would differ; mention thread-safety if relevant.

One-liner to finish (what to say to interviewer): ‚ÄúI‚Äôll use the interleaving approach ‚Äî it runs in O(n) time and O(1) extra space. I‚Äôll first weave copied nodes right after originals, set random pointers using the adjacent copies, then unweave to restore the original list and return the copied head.
