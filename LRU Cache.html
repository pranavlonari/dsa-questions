LRU Cache

Medium BFSGraphLinked ListHash TableArray

Design a data structure that follows the constraints of a Least Recently Used (LRU) cache.

Implement the LRUCache class:

LRUCache(int capacity) Initialize the LRU cache with positive size capacity. int get(int key) Return the value of the key if the key exists, otherwise return -1. void put(int key, int value) Update the value of the key if the key exists. Otherwise, add the key-value pair to the cache. If the number of keys exceeds the capacity from this operation, evict the least recently used key. The functions get and put must each run in O(1) average time complexity.

Example 1:

Input ["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"] [[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]] Output [null, null, null, 1, null, -1, null, -1, 3, 4]

Explanation LRUCache lRUCache = new LRUCache(2); lRUCache.put(1, 1); // cache is {1=1} lRUCache.put(2, 2); // cache is {1=1, 2=2} lRUCache.get(1); // return 1 lRUCache.put(3, 3); // LRU key was 2, evicts key 2, cache is {1=1, 3=3} lRUCache.get(2); // returns -1 (not found) lRUCache.put(4, 4); // LRU key was 1, evicts key 1, cache is {4=4, 3=3} lRUCache.get(1); // return -1 (not found) lRUCache.get(3); // return 3 lRUCache.get(4); // return 4

Constraints:

1 <= capacity <= 3000 0 <= key <= 10^4 0 <= value <= 10^5 At most 2 * 10^5 calls will be made to get and put.

Short script to say to the interviewer

  1. ‚ÄúI‚Äôll implement LRU using a HashMap for O(1) key lookup and a doubly-linked list to maintain usage order (most-recent at head, least-recent at tail).‚Äù
  2. ‚Äúget moves the accessed node to head and returns its value (or -1). put updates or inserts; if inserting and capacity exceeded, remove the tail node and its map entry.‚Äù
  3. ‚ÄúBoth operations are O(1) average time. I can also show a concise LinkedHashMap-based solution if you prefer a one-line library approach.‚Äù

---

Approaches considered (brief)

  1. HashMap + Doubly Linked List (chosen) ‚Äî O(1) get/put, straightforward and classic.
  2. LinkedHashMap (Java built-in) ‚Äî can be used with accessOrder=true and overriding removeEldestEntry for a very short implementation. Simpler but less instructive in interviews.
  3. Ordered array / queue + map ‚Äî not O(1) for updates; not suitable.

I‚Äôll implement HashMap + custom doubly linked list (explicit, interview-friendly).

---

Detailed chosen approach (step-by-step)

  • Data structures:
  • Map map: key ‚Üí node (node holds key, value, prev, next).
  • Doubly linked list with sentinel head and tail nodes: head.next is most recent, tail.prev is least recent.
  • Invariants:
  • List order is recency: newest at head.next ‚Üí oldest at tail.prev.
  • Every key present in map is also in the linked list.
  • Operations:
  • get(key):
  1. If key not in map return -1.
  2. Else retrieve node, move it to head (remove from current position, insert right after head), return node.value.
  3. put(key, value):
  1. If key exists: update node.value, move node to head.
  2. Else: create new node, insert at head, add to map.
  3. If size > capacity: evict tail.prev (least recent): remove from list and map.
  4. Helper methods:
  • remove(node) ‚Äî unlink node from list.
  • addToHead(node) ‚Äî insert node right after head sentinel.
  • Using sentinels simplifies edge cases.

---

Correctness intuition

Moving accessed/updated nodes to head preserves the recency order; eviction from tail removes the least recently used. Map keeps O(1) lookup to the node to perform O(1) moves.

---

Complexity

  • Time: O(1) average for get and put (hashmap ops + pointer updates).
  • Space: O(capacity) for map + linked list nodes.

---

Java implementation (HashMap + Doubly Linked List)

import java.util.*;

public class LRUCache {
    private final int capacity;
    private final Map map;
    // sentinel head and tail
    private final Node head;
    private final Node tail;

    // Doubly linked list node
    private static class Node {
        int key;
        int value;
        Node prev;
        Node next;
        Node(int k, int v) { key = k; value = v; }
    }

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new HashMap<>(capacity * 2);
        // create sentinels
        head = new Node(-1, -1);
        tail = new Node(-1, -1);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        Node node = map.get(key);
        if (node == null) return -1;
        // move accessed node to head (most recent)
        remove(node);
        addToHead(node);
        return node.value;
    }

    public void put(int key, int value) {
        Node node = map.get(key);
        if (node != null) {
            // update value and move to head
            node.value = value;
            remove(node);
            addToHead(node);
        } else {
            // new node
            Node newNode = new Node(key, value);
            addToHead(newNode);
            map.put(key, newNode);

            if (map.size() > capacity) {
                // evict least recently used (tail.prev)
                Node lru = tail.prev;
                remove(lru);
                map.remove(lru.key);
            }
        }
    }

    // unlink node from its current position
    private void remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
        node.prev = null;
        node.next = null;
    }

    // insert node right after head
    private void addToHead(Node node) {
        node.next = head.next;
        node.prev = head;
        head.next.prev = node;
        head.next = node;
    }

    // Optional: small test
    public static void main(String[] args) {
        LRUCache cache = new LRUCache(2);
        cache.put(1,1);
        cache.put(2,2);
        System.out.println(cache.get(1)); // 1
        cache.put(3,3);                   // evicts key 2
        System.out.println(cache.get(2)); // -1
        cache.put(4,4);                   // evicts key 1
        System.out.println(cache.get(1)); // -1
        System.out.println(cache.get(3)); // 3
        System.out.println(cache.get(4)); // 4
    }
}

Alternate concise Java solution (LinkedHashMap)

If the interviewer prefers library usage, Java‚Äôs LinkedHashMap supports access-order iteration which directly models LRU. You override removeEldestEntry to evict when size > capacity.

import java.util.*;

public class LRUCacheUsingLinkedHashMap {
    private final LinkedHashMap map;
    private final int capacity;

    public LRUCacheUsingLinkedHashMap(int capacity) {
        this.capacity = capacity;
        // accessOrder = true -> LRU order by access
        this.map = new LinkedHashMap(16, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry eldest) {
                return size() > LRUCacheUsingLinkedHashMap.this.capacity;
            }
        };
    }

    public int get(int key) {
        return map.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        map.put(key, value);
    }
}

Mention that this is short and correct; however, in interviews you may be expected to explain the internal operations (which the custom implementation demonstrates).

---

Tests / Edge cases to mention

  • Basic sequence from prompt:
  • LRUCache(2) -> put(1,1), put(2,2), get(1) -> 1, put(3,3) -> evict key 2, get(2) -> -1, etc.
  • capacity = 1 behavior (every new put evicts old).
  • Put existing key should update value and mark it as most recent.
  • Access pattern where many gets update recency without changing capacity.
  • Large number of operations up to constraints: ensure O(1) average.

---

Talking points / optimizations

  • Use sentinels (dummy head/tail) to avoid null checks on add/remove ‚Äî cleaner code.
HashMap initial size can be tuned to capacity 2 to reduce rehashes.
  • LinkedHashMap is production-ready and succinct; custom code shows data structure knowledge.
