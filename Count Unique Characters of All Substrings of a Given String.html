Count Unique Characters of All Substrings of a Given String

Let's define a function countUniqueChars(s) that returns the number of unique characters in s.

For example, calling countUniqueChars(s) if s = "LEETCODE" then "L", "T", "C", "O", "D" are the unique characters since they appear only once in s, therefore countUniqueChars(s) = 5. Given a string s, return the sum of countUniqueChars(t) where t is a substring of s. The test cases are generated such that the answer fits in a 32-bit integer.

Notice that some substrings can be repeated so in this case you have to count the repeated ones too.

Example 1:

Input: s = "ABC" Output: 10 Explanation: All possible substrings are: "A","B","C","AB","BC" and "ABC". Every substring is composed with only unique letters. Sum of lengths of all substring is 1 + 1 + 1 + 2 + 2 + 3 = 10 Example 2:

Input: s = "ABA" Output: 8 Explanation: The same as example 1, except countUniqueChars("ABA") = 1. Example 3:

Input: s = "LEETCODE" Output: 92

Constraints:

1 <= s.length <= 105 s consists of uppercase English letters only.

Short interview script

‚ÄúCount each character‚Äôs contribution: every occurrence at index i is unique in exactly (i - prev) * (next - i) substrings, where prev is the previous index of that same character (or -1) and next is the next index (or n). Sum these contributions for all occurrences (use long while summing). This is O(n) time and O(1) extra space for a fixed alphabet.‚Äù

---

Intuition (1‚Äì2 sentences)

A character at position i is unique in a substring iff the substring includes i but includes no other occurrence of that same character. The number of ways to choose left and right endpoints satisfying that equals number of choices of left in (prev, i] times choices of right in [i, next): (i-prev)*(next-i).

---

Complexity

  • Time: O(n + A) where A is alphabet size (here 26), effectively O(n).
  • Space: O(n) total for storing positions (or O(A) if tracking last two indices).
We accumulate in long and cast to int at the end (problem guarantees 32-bit fit).

---

Java implementation (clean & tested)

import java.util.*;

class Solution {
    public int uniqueLetterString(String s) {
        int n = s.length();
        // For uppercase letters A..Z (26). If alphabet were larger, adjust accordingly.
        List[] pos = new ArrayList[26];
        for (int i = 0; i < 26; i++) pos[i] = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            pos[s.charAt(i) - 'A'].add(i);
        }

        long ans = 0L;
        for (int c = 0; c < 26; c++) {
            List list = pos[c];
            if (list.isEmpty()) continue;
            int m = list.size();
            for (int i = 0; i < m; i++) {
                int cur = list.get(i);
                int prev = (i == 0) ? -1 : list.get(i - 1);
                int next = (i == m - 1) ? n : list.get(i + 1);
                ans += (long)(cur - prev) * (next - cur);
            }
        }
        return (int) ans;
    }

    // quick test harness
    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.uniqueLetterString("ABC"));      // 10
        System.out.println(sol.uniqueLetterString("ABA"));      // 8
        System.out.println(sol.uniqueLetterString("LEETCODE")); // 92
    }
}

Alternate notes (if asked in interview)

  • You can do this without storing lists by keeping for each character the last two indices seen and updating contributions on-the-fly in a single pass.
  • For arbitrary/unbounded alphabets, use a Map> instead of fixed 26 buckets.
