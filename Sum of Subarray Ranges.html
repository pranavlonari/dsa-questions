Sum of Subarray Ranges

Medium
ArrayStringTwo PointersStackQueue

You are given an integer array nums. The range of a subarray of nums is the difference between the largest and smallest element in the subarray.

Return the sum of all subarray ranges of nums.

A subarray is a contiguous non-empty sequence of elements within an array.

Example 1:

Input: nums = [1,2,3] Output: 4 Explanation: The 6 subarrays of nums are the following: [1], range = largest - smallest = 1 - 1 = 0 [2], range = 2 - 2 = 0 [3], range = 3 - 3 = 0 [1,2], range = 2 - 1 = 1 [2,3], range = 3 - 2 = 1 [1,2,3], range = 3 - 1 = 2 So the sum of all ranges is 0 + 0 + 0 + 1 + 1 + 2 = 4. Example 2:

Input: nums = [1,3,3] Output: 4 Explanation: The 6 subarrays of nums are the following: [1], range = largest - smallest = 1 - 1 = 0 [3], range = 3 - 3 = 0 [3], range = 3 - 3 = 0 [1,3], range = 3 - 1 = 2 [3,3], range = 3 - 3 = 0 [1,3,3], range = 3 - 1 = 2 So the sum of all ranges is 0 + 0 + 0 + 2 + 0 + 2 = 4. Example 3:

Input: nums = [4,-2,-3,4,1] Output: 59 Explanation: The sum of all subarray ranges of nums is 59.

Constraints:

1 <= nums.length <= 1000 -10^9 <= nums[i] <= 10^9

Key idea (short)

Sum of subarray ranges = sum over all subarrays (max ‚àí min) = (sum over all elements of their contribution as maximum) ‚àí (sum over all elements of their contribution as minimum).

For each element nums[i] we can count how many subarrays have it as the maximum (or minimum). If there are L choices for the left boundary and R choices for the right boundary so that nums[i] is the maximum in the subarray, then it contributes nums[i] L R to the total of maxima. We do the same for minima and subtract.

To compute L and R quickly for every index we use monotonic stacks to find for each index:

  • previous greater element index (strictly greater) and next greater-or-equal element index for maxima counting, and
  • previous smaller element index (strictly smaller) and next smaller-or-equal element index for minima counting.

(The particular choice of strict/ non-strict comparisons handles duplicates correctly and ensures each subarray is counted exactly once.)

This yields an O(n) algorithm using stacks.

---

Correctness intuition about equal elements

When duplicates exist, we must avoid double counting. A common, correct convention is:

  • For maximum contribution of nums[i]:
  • Let prev be the index of the previous element strictly greater than nums[i] (search left).
  • Let next be the index of the next element greater than or equal to nums[i] (search right).
  • Then L = i - prev, R = next - i. Any subarray whose maximum is at index i must start in one of these L positions and end in one of these R positions.
  • For minimum contribution of nums[i]:
  • Let prev be the index of the previous element strictly smaller than nums[i].
  • Let next be the index of the next element smaller than or equal to nums[i].
  • Then L = i - prev, R = next - i for minima counting.

This asymmetric choice (strict on left, non-strict on right) breaks ties consistently and avoids counting the same subarray for multiple equal elements.

---

Complexity

  • Time: O(n) ‚Äî each element pushed/popped at most once for each stack pass.
  • Space: O(n) for stacks and helper arrays.

Java implementation

import java.util.*;

class Solution {
    public long subArrayRanges(int[] nums) {
        int n = nums.length;
        long sumMax = 0L;
        long sumMin = 0L;

        // Arrays to hold indices of previous greater, next greater-or-equal
        int[] prevGreater = new int[n];
        int[] nextGreaterEq = new int[n];

        // Arrays to hold indices of previous less, next less-or-equal
        int[] prevLess = new int[n];
        int[] nextLessEq = new int[n];

        // ---- previous greater (strict) ----
        Deque st = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            while (!st.isEmpty() && nums[st.peek()] <= nums[i]) {
                st.pop();
            }
            prevGreater[i] = st.isEmpty() ? -1 : st.peek();
            st.push(i);
        }

        // ---- next greater-or-equal ----
        st.clear();
        for (int i = n - 1; i >= 0; i--) {
            while (!st.isEmpty() && nums[st.peek()] < nums[i]) {
                st.pop();
            }
            nextGreaterEq[i] = st.isEmpty() ? n : st.peek();
            st.push(i);
        }

        // ---- previous less (strict) ----
        st.clear();
        for (int i = 0; i < n; i++) {
            while (!st.isEmpty() && nums[st.peek()] >= nums[i]) {
                st.pop();
            }
            prevLess[i] = st.isEmpty() ? -1 : st.peek();
            st.push(i);
        }

        // ---- next less-or-equal ----
        st.clear();
        for (int i = n - 1; i >= 0; i--) {
            while (!st.isEmpty() && nums[st.peek()] > nums[i]) {
                st.pop();
            }
            nextLessEq[i] = st.isEmpty() ? n : st.peek();
            st.push(i);
        }

        // accumulate contributions
        for (int i = 0; i < n; i++) {
            long leftMax = i - prevGreater[i];
            long rightMax = nextGreaterEq[i] - i;
            sumMax += (long) nums[i] * leftMax * rightMax;

            long leftMin = i - prevLess[i];
            long rightMin = nextLessEq[i] - i;
            sumMin += (long) nums[i] * leftMin * rightMin;
        }

        return sumMax - sumMin;
    }

    // quick test
    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.subArrayRanges(new int[]{1,2,3}));        // 4
        System.out.println(sol.subArrayRanges(new int[]{1,3,3}));        // 4
        System.out.println(sol.subArrayRanges(new int[]{4,-2,-3,4,1}));  // 59
    }
}

Worked example (brief)

nums = [1,2,3]:
  • Contribution as maxima:
  • 1 is max only in subarrays starting and ending at itself ‚Üí contributes 1 1 1 = 1.
  • 2 is max in subarrays [2], [1,2] ‚Üí 2 2 1 = 4.
  • 3 is max in [3], [2,3], [1,2,3] ‚Üí 3 3 1 = 9.
  • sumMax = 14.
  • Contribution as minima:
  • 1: min in [1], [1,2], [1,2,3] ‚Üí 1 _ 3 _ 1 = 3
  • 2: min in [2], [2,3] ‚Üí 2 _ 2 _ 1 = 4
  • 3: min in [3] ‚Üí 3 _ 1 _ 1 = 3
  • sumMin = 10.

Answer = 14 ‚àí 10 = 4.

---

Notes & pitfalls

  • Use long to accumulate contributions because counts times values can overflow 32-bit (values up to ¬±1e9 and up to ~n¬≤ contributions).
  • The strict/non-strict comparisons must be chosen consistently as shown; otherwise duplicates can be double-counted or missed.
  • The algorithm generalizes to other problems counting subarrays where an element is max/min.
