Maximum Number of Robots Within Budget

Medium
ArrayStringTwo PointersSliding WindowQueue

Problem restatement Given two arrays chargeTimes and runningCosts of length n and a budget, find the maximum length of a contiguous subarray (consecutive robots) such that:

cost(window) = max(chargeTimes in window) + windowSize * sum(runningCosts in window) <= budget

Return the maximum possible window size (or 0 if none).

Input (example 1) chargeTimes = [3,6,1,3,4], runningCosts = [2,1,3,4,5], budget = 25 Output: 3

Input (example 2) chargeTimes = [11,12,19], runningCosts = [10,8,7], budget = 19 Output: 0

Constraints (provided)

  • n = chargeTimes.length = runningCosts.length, 1 <= n <= 5*10^4
  • 1 <= chargeTimes[i], runningCosts[i] <= 10^5
  • 1 <= budget <= 10^15

Key idea (one-liner)

Use a sliding window with two helpers:

  • a monotonic decreasing deque of indices to get max(chargeTimes) in O(1),
  • a running long sum of runningCosts for the current window.

Expand the right end; while the window cost exceeds budget, shrink from the left. Track the maximum valid window size seen.

Cost formula for window [left..right]:

cost = chargeTimes[deque.peekFirst()] + (right - left + 1) * sumRunningCosts

Use long for the sum and cost to avoid overflow.

---

Complexity

  • Time: O(n) amortized ‚Äî each index enters/exits deque at most once and left/right each move up to n times.
  • Space: O(n) worst-case (deque), typically O(k).

---

Java implementation

import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public int maximumRobots(int[] chargeTimes, int[] runningCosts, long budget) {
        int n = chargeTimes.length;
        Deque dq = new ArrayDeque<>(); // indices of chargeTimes in decreasing order
        long sumRunning = 0L;
        int left = 0;
        int ans = 0;

        for (int right = 0; right < n; right++) {
            // add running cost
            sumRunning += runningCosts[right];

            // maintain monotonic decreasing deque for chargeTimes
            while (!dq.isEmpty() && chargeTimes[dq.peekLast()] <= chargeTimes[right]) {
                dq.pollLast();
            }
            dq.offerLast(right);

            // shrink while invalid
            while (left <= right) {
                long windowSize = (long) (right - left + 1);
                long maxCharge = (dq.isEmpty() ? 0L : (long) chargeTimes[dq.peekFirst()]);
                long cost = maxCharge + windowSize * sumRunning;
                if (cost <= budget) break;

                // remove left element
                if (!dq.isEmpty() && dq.peekFirst() == left) {
                    dq.pollFirst();
                }
                sumRunning -= runningCosts[left];
                left++;
            }

            // current window [left..right] is valid
            ans = Math.max(ans, right - left + 1);
        }

        return ans;
    }

    // quick manual test (you can run as a basic check)
    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.maximumRobots(new int[]{3,6,1,3,4}, new int[]{2,1,3,4,5}, 25L)); // 3
        System.out.println(sol.maximumRobots(new int[]{11,12,19}, new int[]{10,8,7}, 19L)); // 0
    }
}

---

Notes & tips

  • Always use long for sumRunning and cost because windowSize * sumRunning may exceed 32-bit.
  • Deque stores indices; when left moves, remove its index from front if it was the maximum.
  • If you prefer, you can also solve with binary search on window size + check using deque (O(n log n) instead of O(n)), but the sliding-window approach is optimal and simpler for this problem.
