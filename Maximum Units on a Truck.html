Maximum Units on a Truck

You are assigned to put some amount of boxes onto one truck. You are given a 2D array boxTypes, where boxTypes[i] = [numberOfBoxesi, numberOfUnitsPerBoxi]:

numberOfBoxesi is the number of boxes of type i. numberOfUnitsPerBoxi is the number of units in each box of the type i. You are also given an integer truckSize, which is the maximum number of boxes that can be put on the truck. You can choose any boxes to put on the truck as long as the number of boxes does not exceed truckSize.

Return the maximum total number of units that can be put on the truck.

Example 1:

Input: boxTypes = [[1,3],[2,2],[3,1]], truckSize = 4 Output: 8 Explanation: There are:

  • 1 box of the first type that contains 3 units.
  • 2 boxes of the second type that contain 2 units each.
  • 3 boxes of the third type that contain 1 unit each.
You can take all the boxes of the first and second types, and one box of the third type. The total number of units will be = (1 _ 3) + (2 _ 2) + (1 \* 1) = 8. Example 2:

Input: boxTypes = [[5,10],[2,5],[4,7],[3,9]], truckSize = 10 Output: 91

Constraints:

1 <= boxTypes.length <= 1000 1 <= numberOfBoxesi, numberOfUnitsPerBoxi <= 1000 1 <= truckSize <= 10^6

Maximum Units on a Truck ‚Äî interview-ready answer

Problem restatement (short): You have several box types described as [count, unitsPerBox]. You can load at most truckSize boxes. Choose boxes to maximize total units loaded. Return that maximum. Key idea (one-liner): Greedy ‚Äî take boxes with the highest unitsPerBox first. Fill as many of that type as possible (up to remaining truckSize), then move to the next best type. Why greedy works: Each box of a given type contributes the same unitsPerBox. To maximize the sum for a constrained number of boxes, always pick the highest-contributing boxes first.

---

Algorithm (steps)

  1. Sort boxTypes in descending order by unitsPerBox.
  2. Initialize remaining = truckSize and totalUnits = 0.
  3. For each type (count, units) in sorted order:
  • take = min(count, remaining)
  • totalUnits += take * units
  • remaining -= take
  • if remaining == 0 break
  1. Return totalUnits.
Time: O(m log m) for sorting (m = number of box types). Space: O(1) extra (or O(m) if sorting copies).

---

Java implementation (clean & safe)

import java.util.*;

public class MaximumUnitsOnTruck {
    public int maximumUnits(int[][] boxTypes, int truckSize) {
        // sort by unitsPerBox descending
        Arrays.sort(boxTypes, (a, b) -> Integer.compare(b[1], a[1]));

        long total = 0L;               // use long to be safe
        int remaining = truckSize;

        for (int[] type : boxTypes) {
            if (remaining == 0) break;
            int count = type[0];
            int units = type[1];
            int take = Math.min(count, remaining);
            total += 1L * take * units;
            remaining -= take;
        }

        // result fits in int by constraints, but cast safely
        return (int) total;
    }

    // quick manual tests
    public static void main(String[] args) {
        MaximumUnitsOnTruck sol = new MaximumUnitsOnTruck();

        int[][] boxes1 = {{1,3},{2,2},{3,1}};
        System.out.println(sol.maximumUnits(boxes1, 4)); // 8

        int[][] boxes2 = {{5,10},{2,5},{4,7},{3,9}};
        System.out.println(sol.maximumUnits(boxes2, 10)); // 91

        int[][] boxes3 = {{1000,1000}};
        System.out.println(sol.maximumUnits(boxes3, 1_000_000)); // 1000 * 1_000_000? clipped by count: 1000*1000 = 1_000_000
    }
}

---

Edge cases / notes

  • If truckSize is larger than the sum of all boxes, you simply take everything.
  • Use long for accumulation to avoid overflow in intermediate multiplication (though final result fits within the problem constraints).
  • Sorting comparator compares units only; if equal-unit types exist, order among them doesn't matter.
