Maximum Length of Subarray With Positive Product

Given an array of integers nums, find the maximum length of a subarray where the product of all its elements is positive.

A subarray of an array is a consecutive sequence of zero or more values taken out of that array.

Return the maximum length of a subarray with positive product.

Example 1:

Input: nums = [1,-2,-3,4] Output: 4 Explanation: The array nums already has a positive product of 24. Example 2:

Input: nums = [0,1,-2,-3,-4] Output: 3 Explanation: The longest subarray with positive product is [1,-2,-3] which has a product of 6. Notice that we cannot include 0 in the subarray since that'll make the product 0 which is not positive. Example 3:

Input: nums = [-1,-2,-3,0,1] Output: 2 Explanation: The longest subarray with positive product is [-1,-2] or [-2,-3].

Constraints:

1 <= nums.length <= 10^5 -10^9 <= nums[i] <= 10^9

---

Short script to say to the interviewer

‚ÄúI‚Äôll split the array by zeros (zeros break any valid subarray). For each zero-free segment I can either (A) count negatives and use positions of first/last negative to compute the best length, or (B) do a single-pass DP tracking the longest prefix with positive product and longest prefix with negative product ‚Äî both are O(n). I‚Äôll implement the single-pass DP because it‚Äôs clean and linear in one sweep.‚Äù

---

Key idea / intuition

  • Any subarray containing 0 has product 0 (not positive). So treat each block between zeros independently.
  • In a zero-free block:
  • If the number of negative elements is even ‚Üí the whole block has positive product (take its length).
  • If odd ‚Üí we must remove a prefix up to the first negative or a suffix starting at the last negative; the maximum positive-product subarray is the longer of those two options.
  • Alternatively (single-pass DP): maintain two lengths while scanning a block:
  • posLen = length of longest subarray ending here with positive product.
  • negLen = length of longest subarray ending here with negative product.
Update them based on current number sign. Reset both at zero when encountering a zero.

This DP approach does exactly what the first idea does but in one pass and without storing indices.

---

Single-pass algorithm (preferred)

Scan nums left‚Üíright, maintaining:

  • posLen ‚Äî length of longest subarray ending at current index with positive product.
  • negLen ‚Äî length of longest subarray ending at current index with negative product.
  • ans ‚Äî global max posLen.

Rules when seeing x:

  • If x == 0: reset posLen = 0, negLen = 0.
  • Else if x > 0: posLen = posLen + 1, negLen = (negLen > 0) ? negLen + 1 : 0.
  • Else (x < 0): swapping effect:
  • newPos = (negLen > 0) ? negLen + 1 : 0
  • newNeg = posLen + 1
  • set posLen = newPos, negLen = newNeg.

After each non-zero index update ans = max(ans, posLen).

This finds the longest positive-product subarray ending at each index and takes the best overall.

---

Complexity

  • Time: O(n) ‚Äî single pass.
  • Space: O(1) ‚Äî constant extra memory.

---

Java implementation (single-pass)

public class MaxLenPositiveProductSubarray {
    public int getMaxLen(int[] nums) {
        int posLen = 0;   // longest ending here with positive product
        int negLen = 0;   // longest ending here with negative product
        int ans = 0;

        for (int x : nums) {
            if (x == 0) {
                posLen = 0;
                negLen = 0;
            } else if (x > 0) {
                posLen = posLen + 1;
                negLen = (negLen > 0) ? negLen + 1 : 0;
            } else { // x < 0
                int newPos = (negLen > 0) ? negLen + 1 : 0;
                int newNeg = posLen + 1;
                posLen = newPos;
                negLen = newNeg;
            }
            if (posLen > ans) ans = posLen;
        }

        return ans;
    }

    // quick tests
    public static void main(String[] args) {
        MaxLenPositiveProductSubarray solver = new MaxLenPositiveProductSubarray();

        System.out.println(solver.getMaxLen(new int[]{1,-2,-3,4}));         // expected 4
        System.out.println(solver.getMaxLen(new int[]{0,1,-2,-3,-4}));     // expected 3
        System.out.println(solver.getMaxLen(new int[]{-1,-2,-3,0,1}));     // expected 2
        System.out.println(solver.getMaxLen(new int[]{0,0,0}));            // expected 0
        System.out.println(solver.getMaxLen(new int[]{-1}));               // expected 0
        System.out.println(solver.getMaxLen(new int[]{5, -1, -1, -1, 2})); // example mixed
    }
}

Alternate (two-phase) approach ‚Äî segment + first/last negative

If you want to explain the index-based method quickly:

  • For each zero-free segment [L..R]:
  • Count negatives. If even ‚Üí candidate length = R-L+1.
  • Else find first negative index f and last negative index l. Candidate length = max(R-f, l-L).
  • Keep max across segments.

This is a good follow-up if interviewer asks for another view.

---

Edge cases to mention

  • Array of length 1 (e.g., [0] ‚Üí 0, [-1] ‚Üí 0, [5] ‚Üí 1).
  • All zeros ‚Üí 0.
  • Large arrays with many sign changes ‚Äî algorithm still O(n).
  • Values up to ¬±1e9 ‚Äî we never multiply them so no overflow concerns.
